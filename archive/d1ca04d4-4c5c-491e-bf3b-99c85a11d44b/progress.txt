## Codebase Patterns

### Backend API Structure (backend/server.js)
- **Framework:** Express.js
- **Response pattern:** Use `res.json()` to send JSON responses
- **Route organization:** All API routes under `/api/*` prefix
- **Error handling:** Return error object with message: `res.status(500).json({ error: err.message })`
- **Test framework:** Vitest with supertest for HTTP testing
- **Test file:** `backend/server.test.mjs` (ESM format)
- **Test pattern:** Use `describe` blocks for grouping, async/await for requests

### Testing Patterns
```javascript
describe('GET /api/endpoint', () => {
    it('returns expected data', async () => {
        const res = await request(app).get('/api/endpoint');
        expect(res.status).toBe(200);
        expect(res.body.field).toBeDefined();
    });
});
```

### Commands
- **Run tests:** `npm test` (runs vitest in backend directory)
- **Typecheck:** `npm run typecheck` (runs tsc --noEmit)
- **Dev server:** `npm run dev` (nodemon)
