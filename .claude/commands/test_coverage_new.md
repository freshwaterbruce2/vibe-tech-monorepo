## Phase 2: Generate Missing Tests (AI-Powered)

Create test files for untested code using AI analysis

**AUTONOMOUS AI TASK** - Generate test files for each untested source file

**For each file** in untested files list:

1. **Read source file** to analyze:
   - Exported functions, classes, components
   - Function signatures and return types
   - Dependencies and imports
   - Edge cases from validation logic
   - Error handling patterns

2. **Generate comprehensive test file** with structure:
   - Import statements (vitest framework)
   - Test suite with describe blocks
   - Happy path tests (valid inputs)
   - Edge case tests (empty, null, boundary values)
   - Error handling tests (invalid inputs, exceptions)
   - Mock external dependencies (APIs, file system, database)

3. **Write test file** using Write tool to same directory as source

4. **Track progress** with TodoWrite:
   - Create todo for each file
   - Mark completed as tests are generated
   - Show progress: "Generated tests: X/Y"

**Quality Guidelines**:

- Aim for 80%+ statement coverage
- Test public API only (exported functions/classes)
- Focus on behavior, not implementation
- Include at least: happy path, edge cases, error cases
- Use vitest syntax: describe, it, expect, vi.fn()
