# monitoring-cycle - System Health Monitoring

Continuously monitor workspace health and automatically fix common issues.

## Safety Checks (Execute First)

Verify monitoring access:

```bash
test -d D:/databases && test -d D:/logs
```

## Phase 1: Health Checks

Scan system components

Execute:
Check databases: ls -lh D:/databases/_.db Check logs: tail -100 D:/logs/_/\*.log | grep ERROR Check services: pm2 list

## Phase 2: Issue Detection

Identify problems

Validation: Categorize: critical, warning, info

## Phase 3: Auto-Remediation

Fix common issues

For each detected issue:

1. Determine fix action 2. Execute fix (restart service, clear space, etc.) 3. Validate fix worked

## Phase 4: Alert

Report status

Execute:
Save report to D:\logs\monitoring-20260118-212533.txt
