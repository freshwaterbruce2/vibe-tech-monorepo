# Project Completion Rules

Priority: CRITICAL — MANDATORY

## Core Principle

Complete features to 100% before moving to the next task. A working finished feature is always better than multiple half-finished ones. Never mark a task complete unless a user can actually use it end-to-end.

## Completion Checklist

Before marking any task complete:

- [ ] Feature fully implemented (no placeholders, no TODOs)
- [ ] Tests written and passing
- [ ] Integrated into the main application (not just written in isolation)
- [ ] User can access and use the feature end-to-end
- [ ] No console errors, no TypeScript errors, no lint warnings
- [ ] Dependencies are current stable versions
- [ ] No regressions introduced
- [ ] Code follows project standards

## Up-to-Date Information

Always verify libraries and patterns are current. Use web search to confirm latest versions and check official docs for breaking changes. Do not assume 2024/2025 knowledge is still valid.

## Workflow

1. Understand requirements fully before starting
2. Implement with production-quality code and tests
3. Verify integration — wire it into the app, not just write the code
4. Clean up: remove debug code, fix TODOs, confirm no regressions
5. Mark complete only when all checklist items pass
