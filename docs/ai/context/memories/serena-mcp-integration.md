# Serena MCP Integration Guide

Last Updated: 2026-01-07

## Overview

Serena is an advanced semantic code analysis MCP server with LSP integration, memory system, and autonomous agent capabilities.

## Key Features

- Semantic code search using Language Server Protocol (LSP)
- Project-specific memory system
- Context-aware assistance (agent mode for autonomous operation)
- Symbol-level code navigation and editing
- Real-time dashboard monitoring: <http://localhost:24282/dashboard/>

## Best Practices (2026)

### 1. Start Read-Only for Safety

```yaml
read_only: true  # Analysis-only mode

# After verification:
read_only: false  # Enable editing
```

### 2. Use Agent Context

```yaml
context: agent  # Autonomous code assistance (recommended)
```

### 3. Performance Optimization

```yaml
default_max_tool_answer_chars: 200000  # Increased for Nx monorepo
tool_timeout: 300  # 5 minutes for complex operations
token_count_estimator: ANTHROPIC_CLAUDE_SONNET_4
```

### 4. Enable Optional Tools

```yaml
included_optional_tools:
  - execute_shell_command  # Run pnpm, nx, git
  - write_memory           # Store insights
  - read_memory            # Retrieve knowledge
  - list_memories          # Browse memories
```

## Available Tools

### Code Analysis

- `find_symbol` - Search symbols by name/path
- `find_referencing_symbols` - Find all references
- `get_symbols_overview` - File structure overview
- `search_for_pattern` - Regex search across files

### Code Editing

- `replace_symbol_body` - Replace function/class body
- `insert_after_symbol` - Add code after symbol
- `insert_before_symbol` - Add code before symbol
- `replace_content` - Regex-based replacement
- `create_text_file` - Create new files

### Navigation

- `list_dir` - Directory listing (respects gitignore)
- `read_file` - Read file contents
- `activate_project` - Switch between projects

### Memory System

- `write_memory` - Store project knowledge
- `read_memory` - Retrieve stored knowledge
- `list_memories` - List available memories
- `delete_memory` - Remove outdated memories

### Thinking Tools (Metacognitive)

- `think_about_collected_information` - Ponder completeness
- `think_about_task_adherence` - Check if on track
- `think_about_whether_you_are_done` - Verify completion

## Common Workflows

### 1. Code Search Workflow

```plaintext
1. find_symbol for known names
2. search_for_pattern for unknown patterns
3. find_referencing_symbols to trace usage
4. get_symbols_overview for file structure
```

### 2. Code Editing Workflow

```plaintext
1. find_symbol to locate target
2. read_file to understand context
3. replace_symbol_body or insert_after_symbol
4. Verify with LSP feedback
```

### 3. Memory Management

```plaintext
1. write_memory after learning important patterns
2. read_memory when context is needed
3. list_memories to browse knowledge
4. delete_memory for outdated information
```

## Troubleshooting

### Issue: "No active project"

**Solution**: `activate_project` tool with path "C:\dev"

### Issue: LSP not working

**Solution**: Verify language configuration in project.yml

```yaml
languages:
  - typescript
  - python
  - rust
```

### Issue: Tool timeout

**Solution**: Increase timeout in global config

```yaml
tool_timeout: 300  # 5 minutes
```

### Issue: Memory not found

**Solution**:

1. Check: C:\dev\.serena\memories\
2. Use: list_memories tool
3. Verify: memory_retention_days in config
